Установка yay
sudo pacman -S --needed git base-devel && git clone https://aur.archlinux.org/yay.git && cd yay && makepkg -si

===Резервное копирование системы===
Создание:
1. Устанавливаем vorta borg (yay -S vorta borg).
2. Запускаем программу, внизу слева нажимаем на плюсик и создаем новый профиль (Create new profile -> придумываем название -> нажимаем save).
3. Во вкладке Repository нажимаем на плюсик -> New Repository -> в открывшемся окне нажимаем на иконку папки и создаем новую папку в директории user (справа сверху иконка Create new folder), в которой будет лежать копия -> нажимаем Open.
4. Вводим пароль 2 раза -> нажимаем Add.
5. Переходим во вкладку Sources -> справа снизу нажимаем на плюсик и выбираем, что хотим добавить в резервную копию -> создаем/добавляем файл/папку в директории user -> нажимаем Open -> нажимаем Start Backup.

Проверка:
Переходим во вкладку Archives -> выбираем копию (галочка) и нажимаем Extract -> создаем папку, в которую будем выгружать копию и нажимаем Open.


===Образ системы===
Нужен сетевой мост и общая папка с ПК!!
Создание:
1. Устанавливаем archiso (sudo pacman -S archiso).
2. В терминале пишем:
cp -r /usr/share/archiso/configs/releng ~/название_директории
cd ~/название_директории
sudo mkarchiso -v -w ./work -o ./out ./

Проверка:
1. В проводнике находим созданный образ (название_директории_из_создания/out) и переносим в общую папку.
2. Открываем virtualbox на ПК и создаем новую машину как обычно, но образ iso выбираем наш только что созданный образ.


===Точка восстановления системы===
Создание:
1. Устанавливаем timeshift (yay -S timeshift).
2. Запускаем программу (sudo timeshift-gtk) -> выбираем RSYNC и нажимаем Next -> выбираем диск и нажимаем Next -> нажимаем Next до конца, а затем Finish.
3. Нажимаем на кнопку Create.

Проверка:
ПКМ на точку восстановления и нажимаем View Rsync Log for Create.


===Группы пользователей===
1. Создание пользователя (без прав суперпользователя)
sudo useradd -m -s /bin/bash имя_пользователя
sudo passwd имя_пользователя -> Enter -> пароль  

Также пользователя можно создать в настройках: System -> Users.

2. Создание группы
bash
sudo groupadd название_группы
3. Добавление пользователя в группу
sudo usermod -aG название_группы имя_пользователя
4. Проверка групп пользователя
groups имя_пользователя


===Настройка аутентификации и авторизации===
Проверка наличия второго пользователя и его прав

1. На экране блокировки выбираем второго пользователя, вводим пароль
2. В терминале проверяем наличие у него прав суперпользователя:
sudo su -> вводим пароль
Должно отобразиться сообщение "имя_пользователя is not in the sudoers file."


===Журнал мониторинга===
Проверка активности журнала мониторинга логов:
systemctl status systemd-journald

Проверка логов текущей загрузки:
journalctl -b

Проверка логов конкретной службы (например, SSH)
journalctl -u sshd
